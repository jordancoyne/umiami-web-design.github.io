How to download your notes

You will need to have GitHub Desktop installed on your computer. If you have not read Getting Started, please do that first.

  1. Click the "Clone in Desktop" icon in the course-notes repository.
  2. If you receive an alert asking you to launch GitHub Desktop, click "Launch Application".
  3. When prompted to save the repository, click "Clone". Be sure to note the directory path where you are saving this on your computer.

Troubleshooting

If you are re-directed to the GitHub Desktop web site when you attempt to clone a repository, log out of GitHub Desktop (login details are under GitHub Desktop > Preferences), then log in again. Do the same with your GitHub account on the web. Repeat the steps above.

How to update your notes

If you "watch" this repo on GitHub, every time the notes have been changed or updated, a notice will appear on your main page feed. When this happens:

  1. Open GitHub Desktop.
  2. Select the "notes" repository in the left column.
  3. Click "Sync" in the top, right corner.

The most recent set of notes (and any updates to previous notes) will be downloaded to your computer.

How to review your notes

All of the notes are written in HTML (and CSS). This is to help become familiar with the code you will be writing, in addition to learning the content contained within the notes themselves.

There are two ways you can review the notes:

  1. View the notes as they are formatted for the web. Open one of the files in Google Chrome (or another web browser) by selecting File, then Open File. This is the best way to review slides you may have missed in class so you can supplement your handwritten notes.
  2. View the text and code used to create the .html documents. Open one of the files in your text editor. You can do this by right-clicking (ctrl-click) on the notes repo in GitHub Desktop and select Open in Atom (if you are using Atom). If you are using another text editor, select Open in Finder, then drag-and-drop the files onto your text editor's icon. You can also open the files through the text editor's Open menu.

Having access the notes in this format also gives you something to tinker with.

The more code you see and play with, the more comfortable you should become.

Frequently Asked Questions (FAQ)

Can I view them online? What is the URL for these notes?

No.

I purposely do not set up as a web site so you will download them and play with the code. If the pages were easily accessible, no one would bother looking into what makes them work.

I want you get familiar with viewing code using a web browser and a text editor. See how the pages look in different browsers like Firefox, Safari and Opera. Can you spot differences? Try making changes to the code.

What if I break the code?

That is okay! Do it! Break it! You will learn more from breaking and fixing than doing everything perfect the first time.

If you break the code, you can always revert to a working copy.

To do so, open GitHub Desktop and select the "notes" repository in the left column. Next, click the tab which displays the text uncommitted changes. This will reveal a list of files which have been changed.

Select the files you want to restore. Select Repository from the menu, then Discard Changes to Selected Files.

This will restore the "broken" file(s) to a working version.
