Introduction

To prepare for this course, you must have your computer set up before the second week of class. We begin coding that week.

Computers are very unforgiving when it comes to instructions. As the saying goes, "Garbage in, garbage out." Poor input leads to bad output — and a frustrated user!

As with everything in this course, read these instructions carefully and follow them step-by-step.

Get your computer a checkup

Students who take this course often own computers in disarray. Operating system software is outdated. Malware and viruses have made the device slow. Files are not backed up.

If this sounds like your machine, visit the Student Technology Help Desk. You should have them:

Back up your hard drive. Technology is not immune to failure. Back up your important or sensitive files to an external hard drive or cloud service. Get in the habit of making weekly -- or better yet, daily -- backups.

Update your operating system. Keeping your operating system up-to-date can improve performance and security. Updates may take a while to download if you are not on a high-speed Internet connection. It is best to update your software while on campus.

Make SecureCanes your default Wi-Fi connection. All individuals affiliated with the university should be using this access point as the default for security purposes since it offers authenticated encryption.

Installing Software

Google Chrome

Google Chrome, the most popular browser in use worldwide, is the only browser we use in this course. It supports up-to-date web standards and offers a suite of web development tools.

Do not use other web browsers for this course. Only use Chrome.

Web Developer Toolbar (for Chrome)

This extension offers simple tools which can give you information about your web page or help in the debugging process.

uBlock (for Chrome)

uBlock is an open-source ad blocker. It will disable ads and third-party trackers. Using an ad blocker may actually improve your browsing experience.

(We can debate the merits/transgressions of ad blockers another time. Considering ads and trackers can slow down our computers, we are using a blocker to make our process more efficient.)

A text/code editor

Options include: Atom or Brackets.

To code, you need a plain text editor. These are different from word processors (i.e. Microsoft Word, Apple's Pages).

Your textbook (HTML & CSS, pp. 29-32) mentions Mac OS X's TextEdit and Windows' Notepad as options. These text editors will work, but Atom and Brackets are optimized for coding.

I use Atom during lectures, but some students prefer Brackets. Choose an editor your feel comfortable with.

Spend time getting familiar with your code editor. Play with menu options to learn basic features. See if the official site offers an introduction or walkthrough. Read the editor's documentation.

GitHub Desktop

See below.

Create a GitHub account

GitHub is a (free) web site where developers store open-source code. Users can also host static web pages, which we will create in this course. You will submit assignments projects on GitHub.

Make sure you verify your email address. Fill out your profile. Add a photo. :)

Once your account and profile are set, practice by getting a project on GitHub. (Keep it simple. Try uploading a plain text file.)

GitHub's help documentation is extensive and explain the basics of how to use the site.

Install GitHub Desktop

GitHub Desktop is an application which allows you to track changes to your code (using version control software called git) and post your code to GitHub.

git is a very powerful (and helpful!) tool when first learning to code. It allows you to 'lock down' code, so if you make a mistake which breaks your web page, you can 'roll back' to the last working version.

[Optional] GitHub Freebies!

If you register with your miami.edu e-mail address, you are eligible for GitHub's Student Developer Pack, which offers tools and web services which normally cost hundreds of dollars.

GitHub can be slow in granting access to the Student Developer Pack, so you may want to register sooner than later.

A couple offers may be of help in this course (i.e. a free domain name), but many of the offers are beyond the scope of this course. They may be helpful for those who want to continue building sites in the future.

Hey, it's free. Do you really need another reason? :)
