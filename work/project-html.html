Instructions

Create an HTML version of your résumé using valid HTML5 and best practices.

You have the freedom to be creative in how you present your résumé. It should take into account which part(s) of your résumé are the most important content-wise, be marked up in a way which describes the content and organized using HTML5 structural tags.

This assignment is worth 15 points.

Suggestion: Follow instructions. When you finish your first attempt at coding, print this page and mark off each requirement you have met. Fix your code/résumé to meet the requirements. This will provide you with assurance you have completed what is being asked.

Before You Begin

If you have not created a résumé or need advice creating an effective one, meet with an advisor. Toppel Career Center (5225 Ponce De Leon Blvd, Coral Gables, Fla.) offers walk-in résumé advising, Monday-Friday, 9 a.m.-4:30 p.m. Appointments are also available.

Your résumé is a reflection of you. Make sure what you post to the web leaves a good first impression on anyone who comes across it.

Requirements

We will be looking for a well-structured document, including:

  • Valid HTML (checked using the W3C HTML Validator)
  • Presence of HTML5 structural tags (i.e. header, nav, main, section, article, footer) as needed
  • Proper use of heading (h1-h6) tags
  • Properly encoded HTML entities (where necessary)
  • Use of appropriate tags for marking up respective content
  • Does not use presentational elements (i.e. b, i, hr, etc.)
  • Does not use ornamental characters (i.e. bullets, pipes, etc.)
  • Does not use generic HTML tags (i.e. span, div)

To demonstrate understanding of hypertext, the document must include:

  • at least two external links
  • a navigational (nav) element which provides links to each section of the résumé

In general, your document should:

  • follow best practices as specified during lectures
  • be free of spelling, grammatical and style errors
  • be free of broken links
  • be free of broken images

Optional

You may include an image or photo, but it is not required. If you choose to include an image (social media icons, for instance):

  • you must have the rights to use it
  • it must be included in your GitHub repo (no linking to an image or file hosted on another site)

Submission

By now you should have all required software installed on your computer and familiar with GitHub and GitHub Desktop. You should also have created a repository to host your web pages (username.github.io).

Only one file — resume.html — is required for this assignment. You are to commit your résumé as resume.html to your username.github.io repo.

Create an issue in the project-html repository with your full name and a link to your web page (username.github.io/resume.html) by the deadline. If you are unsure of how to submit, check the example provided in the Issues section.

Do not wait until the last minute to submit your URL.

Grading

One quarter of a point (0.25) will be be deducted for each missing requirement as stated above. One quarter of a point (0.25) will be be deducted for each spelling, grammar or style error.

A missed deadline (submitting the assignment, for instance, at 3:00:01 p.m.) will result in a two (2) point deduction for every day the assignment is late.

Failure to submit the assignment to GitHub as detailed above will result in a five (5) point deduction. This includes an incorrect repo name and/or file name.

(I use a computer script to clone and do an initial check of every student's work. If the script breaks, it means I have to do extra work to grade your assignment. Be thankful I don't grade incorrect submissions as a zero; better yet, just correctly submit your projects.)

Plagiarism or lapses in web ethics, as detailed in the course syllabus, will result in a zero (0) on the assignment, PLUS a 15-point penalty on your final grade and referral to the University of Miami Honor Council.
