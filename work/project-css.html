Instructions

Using a corrected version of your HTML résumé, create a styled version for the web and print using CSS.

You have the freedom to be creative in how you present your résumé. It should demonstrate proper visual design techniques, utilize an organized site architecture and adhere to best practices.

This assignment is worth 15 points.

Suggestion: Follow instructions. When you finish your first attempt at coding, print this page and mark off each requirement you have met. Fix your code/résumé to meet the requirements. This will provide you with assurance you have completed what is being asked.

Requirements

By now, you should have taken the time to correct the HTML version of your résumé based on feedback from your professor.

We will (still) be looking for the requirements of a well-structured HTML document set forth the first project, including:

  • Valid HTML (checked using the W3C HTML Validator)
  • Presence of HTML5 structural tags (i.e. header, nav, main, section, article, footer) as needed
  • Proper use of heading (h1-h6) tags
  • Properly encoded HTML entities (where necessary)
  • Use of appropriate tags for marking up respective content
  • Does not use presentational elements (i.e. b, i, hr, etc.)
  • Does not use ornamental characters (i.e. bullets, pipes, etc.)

To demonstrate proficiency in site architecture, your files should:

  • Be organized using directories (i.e. a css directory for .css files, img directory for images)
  • Be named according to best practices for portability among file systems

To demonstrate understanding in CSS, visual and interactive design principles, your page should:

  • Display well in Google Chrome
  • Exhibit use of box model properties (i.e. margin, border, padding)
  • Demonstrate two positioning techniques (i.e. center a containing element, floats, position, etc.)
  • Include two, well-paired web fonts from a provider such as Google Fonts
  • Set colors for different link "states" (i.e. default, active, visited, hover)
  • Keep content from stretching beyond 960 pixels
  • Avoid "design sins" (see below)

To show an understanding of designing for different media, your project should incorporate a style sheet which makes your résumé appear as if it were created exclusively for print. It should:

  • Use a serif typeface
  • Use print units (points, inches, centimeters, millimeters, etc.)
  • Use only black and white
  • Eliminate unnecessary interactive or multimedia elements

In general, your document should:

  • follow best practices as specified during lectures
  • be free of spelling, grammatical and style errors
  • be free of broken links
  • be free of broken images

If you choose to include an image (a background or social media icons, for instance):

  • you must have the rights to use it
  • it must be included in your GitHub repo (no linking to an image or file hosted on another site)

Design Sins

Your designs (web and print) should avoid the following:

  • Things that blink
  • Warped photos
  • Naked photos (photos lacking a border)
  • Bulky borders, boxes
  • Bad bullets
  • Cheated margins
  • Poorly aligned elements
  • Centering everything
  • Clutter
  • Trapped white space
  • Busy backgrounds
  • Tacky, mismatched or illegible type
  • Mismatched colors

Create an issue in the project-css repository with your full name and a link to your web page (username.github.io/resume.html) by the deadline. If you are unsure of how to submit, check the example provided in the Issues section.

One quarter of a point (0.25) will be be deducted for projects that do not meet the presentational requirements. Failure to attempt a print style sheet will result in a two (2) point deduction.

One half point (0.5) will be deducted for projects that do not meet the structural requirements. One half of a point (0.5) will be be deducted for each spelling, grammar or style error.

You received feedback and had time to correct the errors in project one. Do not make the same mistakes again because now they will hit your grade twice as hard.

A missed deadline (submitting the assignment, for instance, at 3:00:01 p.m.) will result in a two (2) point deduction for every day the assignment is late.

Failure to submit the assignment to GitHub as detailed above will result in a five (5) point deduction. This includes an incorrect repo name and/or file name.

(I use a computer script to clone every student's work. If the script breaks, it means I have to do extra work to grade your assignment. Be thankful I don't grade incorrect submissions as a zero; better yet, just correctly submit your projects.)

Plagiarism or lapses in web ethics, as detailed in the course syllabus, will result in a zero (0) on the assignment, PLUS a 15-point penalty on your final grade and referral to the University of Miami Honor Council.

Tips

  • Sketch out how you want your résumé to look before starting. This will provide you with a visual reference as you code.
  • Start styling from the top of your HTML document. Create the styles which will be inherited (body). Move on to the header, then nav ...
  • Depending your design, you may have to add or modify HTML. You may decide to re-do the entire HTML document. This is okay. Make sure any change you make to the HTML validates before continuing with styling.
  • If you are unsure where you left off with your GitHub repo (username.github.io), your can always delete the repository from your computer and re-clone it from GitHub. Make any necessary changes to resume.html, commit the files, then push to GitHub.
