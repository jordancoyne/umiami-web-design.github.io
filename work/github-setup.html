What is Git? What is GitHub?

git is version control software. It tracks changes you make to a file. Using git while doing web design is helpful because it forces you break down the process into manageable steps and logically group changes.

When you make a mistake, you can 'roll back' to the last step when you know everything worked. For this reason, you should only "commit" changes in which code works.

GitHub is a web site where people can store open-source code for free. GitHub can also serve as a host for simple, static web sites, which is what we create for in class.

We will demonstrate how you can incorporate git into your workflow during a live coding session. For now, become familiar with the basic process of committing changes and pushing them to GitHub.

If the terminology used through this assignment seems confusing, please refer to the GitHub Glossary. Important terms are also linked throughout these instructions. You may also want to consult GitHub's extensive help section and GitHub Desktop's user guides and FAQ.

Want a warm-up? Try GitHub's walkthrough, Getting your project on GitHub or the built-in tutorial in GitHub Desktop.

Part I: Working with GitHub & GitHub Desktop (5 points)

  1. In the top right corner of the github-setup repository (repo), click 'Fork'. This will create a copy of the github-setup repository in your GitHub account. After GitHub redirects you to the forked copy, you should see: username/github-setup (where username is your username) and a message below which says "forked from umiami-web-design/github-setup".
  2. Clone this forked repository to GitHub Desktop. To the left of "Download ZIP" is a button which looks has an arrow pointing down on a computer screen. Click this to clone the repository using GitHub Desktop. (If you receive an alert, "External Protocol Request," click "Launch Application.") When GitHub Desktop appears and prompts you "Clone", do so. Make sure you note where you are saving this repository on your computer.
  3. To access the repository you just cloned, you may navigate through the directories using your computer's Finder. You can also access the file by right-clicking (ctrl-click) on github-setup in the left column. Choose "Open in Finder." (If you are using Atom, try choosing "Open in Atom.") Open q-and-a.txt. Answer the questions using your text editor. Save the file.
  4. Return to GitHub Desktop. You should see a note at the top which reads "1 uncommitted change". The panel just right of the left column shows which files have changed (in this case, q-and-a.txt) with a checkbox next to it. The panel on the right side of GitHub Desktop shows the changes made to q-and-a.txt. When you add to a file, GitHub Desktop displays these changes in green; when you delete, changes are pink.
  5. Below the list of files changed, fill in the "summary" field. It can be something simple such as, "Answered questions." Always make your commit summary descriptive. ("Made a change" isn't very helpful to someone who has cloned a repo of your writing, code, etc.) The "description" field is optional, but can be used to provide more detail if necessary. If everything is set — the answers in your file are correct, the summary is descriptive — click "Commit to master." This locks the changes in place.
  6. At this point, you could make more changes to files, commit them, make more changes and commit them... or, you can push the committed changes to GitHub. Click "Sync" in the top right corner. (If you receive a message to allow your computer access to a keychain, click "Allow.")
  7. Return to your web browser and view the github-status repository you forked into your GitHub account. (This may still be displayed in your browser; if so, just refresh the page.) You should see the q-and-a.txt file has been updated. If you select it, you will see the changes.
  8. In the username/github-setup repository, above the file listing a similar message should be present: "This branch is 1 commit ahead of umiami-web-design:master." This means you have made a change which has not yet been picked up by the (my) master github repository you made a fork of. Click the link next to the message which reads "Pull Request". This will take you a page which allows you to compare your updated version with the master version. If everything appears correctly — again, the answers in your file are correct, the summary is descriptive — click "Create pull request." (You may be prompted to modify your commit message; simply click "Create pull request" again.)
  9. Notice you have been re-directed to the umiami-web-design/github-status repository. If you click on the pull request icon in the right column, you should see a list of pull requests, including your own. This means you have successfully submitted Part I of this exercise!

Part II: Set up your web site (5 points)

  1. On GitHub, create a new repository. Name it username.github.io where username is your GitHub username. Type the username exactly as it appears as GitHub is case sensitive. Click the checkbox next to 'Initialize this repository with a README'.
  2. Click the clone in desktop icon (the same one you used to clone the repository in step one). When prompted to clone the repository, make sure "Clone As:" is username.github.io (again, where username is your username). Make note of the directory you are saving this cloned repo in. You will need to know this later.
  3. Access the cloned repository on your computer. Open the directory (or README file) in Atom. This should get you set up to work in the correct directory.
  4. Create a simple web page and save it as index.html. Remember to include all parts of a proper web page: doctype, html, head, meta, title and body tags.
  5. Add your name wrapped in an h1 tag and a p tag with a short introduction, something like "Hi, this is my web page." It does not need to be complex.
  6. Remember the octocat.png file from the github-setup repository? Move this file from the github-setup directory into the username.github.io directory on your computer. Add the octocat.png to index.html using an img tag. (Do not forget the alt attribute!)
  7. As you did in the first part of this exercise, commit the changes using GitHub Desktop. You should be committing two files: index.html and octocat.png. GitHub Desktop should recognize there are "uncommitted changes". Provide a summary for the changes. Click "Commit to master". Then push the changes to GitHub by clicking "Sync."
  8. Return to your web browser and view your username.github.io repository. The files should appear.
  9. Open a new tab in your web browser and visit http://username.github.io (again, where username is your username). Your simple web page should appear.

That's it! You have published your first web page!
